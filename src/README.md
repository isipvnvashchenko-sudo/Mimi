# Habit Tracker (Трекер привычек)

Проект представляет собой консольное приложение (CLI) для отслеживания ежедневных привычек, разработанное в рамках полного цикла разработки с использованием методологии **TDD (Test-Driven Development)**, ручного приёмочного тестирования и симуляции процесса технической поддержки.

## 1. Архитектура и структура проекта

Приложение построено по многослойной архитектуре с четким разделением ответственности:

```text
habit_tracker/
├── data/
│   └── habits.json      # Хранилище данных в формате JSON
├── src/
│   ├── __init__.py
│   ├── db.py            # Сервисный слой: Чтение и запись данных (JSON)
│   ├── services.py      # Бизнес-логика: Валидация, расчет серий, CRUD
│   └── main.py          # Консольный интерфейс пользователя (CLI)
├── tests/
│   ├── __init__.py
│   └── test_services.py # Модульные тесты (pytest)
├── README.md            # Документация архитектуры и инструкция
├── TEST_CHECKLIST.md    # Чек-лист приёмочного тестирования
├── SUPPORT_LOG.md       # Журнал обращений техподдержки
└── REPORT.md            # Итоговый отчёт и ретроспектива
